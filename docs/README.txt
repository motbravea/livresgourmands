## Documentation

Le dossier `docs/` contient l’ensemble des éléments nécessaires à la compréhension et à la validation du projet.

### Contenu

- `screenshots/`  
  Ce dossier regroupe les captures d’écran des tests réalisés avec Postman.  
  Elles démontrent :
  - l’inscription et l’authentification des utilisateurs
  - l’accès aux routes protégées avec JWT
  - la gestion des rôles (administrateur, client, etc.)
  - les opérations CRUD sur les catégories et les ouvrages

- `data_dictionary.md`  
  Ce document décrit la structure de la base de données :
  - les tables
  - les colonnes
  - les types de données
  - les relations et contraintes

- `rapport.pdf`  
  Ce fichier contient le rapport du projet, incluant :
  - la description générale du système
  - les choix technologiques (Node.js, Express, MySQL)
  - la modélisation de la base de données
  - l’implémentation de la sécurité (JWT, bcrypt)
  - les règles de gestion appliquées
  - les limites et pistes d’amélioration

### Objectif

Cette documentation permet de :
- faciliter la compréhension du projet
- démontrer le bon fonctionnement de l’API
- répondre aux exigences du travail demandé

Elle complète l’implémentation backend et constitue une preuve du travail réalisé.
